# ADR-003: Разделение Quiz и QuizSession

## Статус

Принято

## Контекст

В приложении есть два разных понятия:

- шаблон квиза
- конкретный live-запуск квиза

На первый взгляд можно было бы хранить всё в одной сущности `Quiz`: вопросы, участников, текущий вопрос, статус игры, ответы и результаты.

Но такой подход быстро смешивает две разные ответственности:

- `Quiz` как редактируемая заготовка
- `Quiz` как активная игровая сессия

Например, один и тот же квиз может быть запущен несколько раз:

```text
Quiz: Java Basics

QuizSession #1:
- room_code = ABC123
- участники: Pavel, Anna
- статус: FINISHED

QuizSession #2:
- room_code = XYZ789
- участники: Ivan, Maria
- статус: WAITING
