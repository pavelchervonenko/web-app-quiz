# ADR-001: Monorepo-структура проекта

## Контекст

Проект состоит из нескольких частей:

- backend-приложение на Java 21 и Spring Boot
- frontend-приложение на React и Vite
- конфигурация деплоя и инфраструктуры
- документация проекта и архитектурные решения

Для MVP backend и frontend разрабатываются вместе и тесно связаны общими продуктовыми требованиями, API-контрактами, настройками деплоя и процессом релиза.

Рассматривались два основных варианта:

1. Использовать отдельные репозитории для backend и frontend.
2. Использовать один репозиторий с monorepo-структурой.

Отдельные репозитории дают более сильную изоляцию, но увеличивают сложность координации для небольшого MVP-проекта. В таком случае пришлось бы отдельно настраивать issue tracking, CI/CD и синхронизацию изменений между frontend и backend.

Monorepo позволяет хранить всю структуру приложения в одном месте и упрощает разработку, review и деплой связанных изменений.

## Решение

Использовать monorepo-структуру для проекта.

Репозиторий будет содержать:

```text
/
├── backend/
│   └── Spring Boot backend-приложение
├── frontend/
│   └── React frontend-приложение
├── docs/
│   └── документация проекта и ADR
├── docker-compose.yml
│   └── конфигурация production-like деплоя
└── .github/
    └── CI/CD workflows
